
root:    /api/v1                       /api wird für das forwarding verwendet, /v1 ist für die Versionierung der api

Beispiel : /api/v1/auth/login
            /api/v1/auth/logout

In den nachfolgenden Bespielen wir die root router nicht mehr erwähnt

Ohne gültigen Web Token
************************

/auth
*****
post     /auth/login                   login an user
get      /auth/logout                  logout a user
post     /auth/register                register a new user profile

Nur mit gültigen Web Tocken
***************************

/auth
*****
get      /auth/profile                 get the current logged in profile
put      /auth/profile                 updates the current logged in profile
delete   /auth/profile                 delete the current logged in profile

/users
******
get      /users/{id}/meetups                   gets all meetups for the user
get      /users/{id}/meetup-requests           gets all meetup-requests for the user
get      /users/{id}/meetup-requests/{status}  gets all meetup-requests for the user with the given status

/meetups
********
get      /meetups                      get a list of all meetups
post     /meetups                      create a new meetup
get      /meetups/{id}                 get meetup details
delete   /meetups/{id}                 delete a meetup
put      /meetups/{id}                 updates an existing meetup
get      /meetups/{id}/meetup-requests gets all meetup-requests for a meetup
get      /meetups/{id}/chats           gets all chat-entry for a meetup
post     /meetups/{id}/chats           creates a new chat-entry for a meetup
delete   /meetups/{id}/chats{id}       delete a chat-entry from a meetup

put      /meetups/search               get a list of all meetups based on search criteria (SearchDto)

/meetup-requests
****************
post     /meetup-requests              creates a new meetup-request
get      /meetup-requests/{id}         get meetup-request details
put      /meetup-requests/{id}         updates an existing meetup-request
delete   /meetup-requests/{id}         delete a meetup-request

/halls
******
get      /halls                        get a list of all halls
get      /halls/{id}                   get hall details

search
******
Der search wird mit der Route /meetups/search umgesetzt. Dem Post ist ein SearchDto zu übergeben, welches die such
Kriterien enthält.
